Integrate the Hospital Pages Function within the PetSOS app to display detailed, owner‑friendly profiles for each 24‑hour animal hospital. The feature should be fully embedded under the PetSOS app namespace (e.g., /app/hospitals/[slug]) and reuse existing app theming, authentication, and analytics systems.

Core Objective

Provide pet owners with:

Essential hospital details (name, address, region, 24/7 status)

Owner‑centric service breakdowns (ICU, CT, triage policy, insurance, etc.)

Reference consultation fees (Day / Evening / Night/PH)

Quick contact actions (Call / WhatsApp / Maps)

Accuracy & Transparency — last verified date, and “Report incorrect info” button.

Function Path & Integration

Namespace: /app/hospitals (under PetSOS)

Routes:

/app/hospitals → list view (filter by region, open 24/7, services)

/app/hospitals/[slug] → detail view (full page)

Parent app modules reused:

AuthContext (for user/member login)

UI Kit (buttons, badges, modals)

Analytics (PostHog event tracking)

I18nContext (EN/繁 switching)

ReportModal (for incorrect info)

Tech Stack

Next.js 14 + TypeScript + Tailwind (PetSOS design system)

Prisma + PostgreSQL (extend PetSOS DB)

Supabase Storage / Cloudflare R2 for photos

next-auth roles (admin, hospital_editor, user)

API Endpoints (under PetSOS API namespace)

GET /api/app/hospitals?region=HKI&services=ER,CT — list view.

GET /api/app/hospitals/[slug] — single hospital detail.

PUT /api/app/hospitals/[id] — admin update.

POST /api/app/hospitals/ingest-update — Typeform/Google Form payload ingestion.

POST /api/app/hospitals/[id]/report — report incorrect info.

All APIs inherit existing PetSOS authentication middleware.
